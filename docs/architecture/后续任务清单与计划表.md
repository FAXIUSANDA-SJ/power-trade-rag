# 后续任务清单与计划表

## 1. 当前阶段目标

- 当前目标是完成“架构优化可验收版”。
- 优先收口任务化摄取、OCR 联调、前端状态闭环、配置治理与基础运维能力。
- 暂不进入更重的混合检索、复杂 Agent、结构化数据查询深水区。

## 2. 剩余轮次清单

### 第 1 轮：任务链路闭环

- 统一上传受理、处理中、完成、失败响应语义。
- 补任务错误详情 DTO。
- 补任务状态自动刷新。
- 补文档与任务联动展示。
- 收口删除文档与任务状态的一致性。

### 第 2 轮：OCR 真联调

- 对接真实 OCR HTTP 服务。
- 适配响应字段、鉴权和错误码。
- 增加大文件限制与降级策略。
- 增加 OCR 调试响应摘要。
- 区分 OCR 失败与普通解析失败。

### 第 3 轮：摄取链路增强

- 增加 Worker 并发数配置。
- 增加失败退避与重试策略。
- 增加更强的幂等键与重复提交保护。
- 增加文档重建索引能力。
- 补任务状态流转说明。

### 第 4 轮：前端闭环与验收

- 增加任务统计卡片。
- 增加任务详情和失败原因展示。
- 增加上传后自动刷新或轮询。
- 增加 OCR 联调入口或调试面板。
- 输出联调文档、验收清单和回归项。

### 第 5 轮：检索能力增强

- 页面化检索参数。
- 接入 Rerank 开关和参数。
- 差异化知识库检索策略。
- 为 Milvus + OpenSearch 预留接口。

### 第 6 轮：运维与治理

- 补任务、检索、模型耗时监控。
- 补配置发布记录和回滚说明。
- 补审计日志与异常告警。
- 补任务清理与归档策略。
