# 架构验收清单

## 1. 验收范围

- 文档摄取任务化链路
- Worker 自动消费与失败退避重试
- OCR HTTP provider 联调能力
- 文档管理页与任务中心页状态闭环
- 配置注入与环境变量模板

## 2. 后端验收项

- `上传文档` 返回任务受理响应，而不是同步处理结果
- `ingest-task` 列表、详情、统计、重试接口可用
- 失败任务会记录结构化错误信息
- Worker 对可自动重试错误按退避策略执行
- Worker 对不可自动重试错误进入阻塞状态，不会无脑重试
- OCR 调试接口 `POST /api/document/ocr/test` 返回结构化结果

## 3. 前端验收项

- 文档管理页可看到摄取状态和任务详情
- 文档管理页可直接使用 OCR 调试面板
- OCR 调试面板可展示：
  - `provider`
  - `success`
  - `errorType`
  - `errorCode`
  - `httpStatus`
  - `durationMs`
  - `responsePreview`
  - `extractedText`
- 任务中心页可展示：
  - 任务状态
  - 重试次数
  - 自动重试是否允许
  - 下次自动重试时间
  - 退避状态
  - 阻塞原因

## 4. 配置验收项

- `.env.example` 已包含 OCR 和 Worker 相关配置项
- `application.yml` 与 `.env.example` 中的 OCR / Worker 配置名称一致
- 修改环境变量后，无需改代码即可切换 OCR endpoint、鉴权头和字段映射

## 5. 手动回归步骤

1. 启动后端：`mvn spring-boot:run`
2. 启动前端：`cd power-trade-rag-web && npm run dev`
3. 上传一个普通文档，确认返回“文档已受理”
4. 打开 `任务中心`，确认任务进入 `pending/running/success`
5. 构造一个可失败场景，确认：
   - 失败任务有错误编码
   - 自动重试状态正确
   - 下次自动重试时间可见
6. 在 `文档管理` 中执行 OCR 测试，确认结构化结果展示正常

## 6. 通过标准

- 后端编译通过：`mvn -q -DskipTests compile`
- 前端构建通过：`cd power-trade-rag-web && npm run build`
- OCR 联调至少覆盖 1 个成功样例和 1 个失败样例
- 任务中心至少能看到 1 个失败任务的阻塞原因或退避状态

## 7. 已知剩余项

- 当前“不可自动重试”仍以异常类型和部分兼容性规则共同判定，后续可继续升级为更严格的异常码体系
- 任务链路暂未补充专门的审计日志、告警和归档能力
- 还未提供文档重建索引入口
