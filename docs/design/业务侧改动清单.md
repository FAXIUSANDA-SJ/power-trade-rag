# 业务侧改动清单

## 1. 文档说明

- 本文档只整理业务侧后续改动范围。
- 当前不实施业务代码改动。
- 待架构设计、基础设施方案和技术链路验收后，再按批次启动。

## 2. 业务侧改动范围

### 2.1 问答结果形态调整

- 规则类问题：回答优先返回结论 + 依据条款 + 生效时间。
- 流程类问题：回答优先返回步骤 + 注意事项 + 适用条件。
- 案例类问题：回答优先返回案例摘要 + 适用边界 + 风险提示。
- 未命中问题：返回澄清建议，而不是笼统兜底。

### 2.2 来源与版本展示

- 在聊天页面展示引用来源。
- 显示文档标题、片段、发布时间、生效状态。
- 对存在多个版本的规则，明确“当前生效版本”和“历史版本”。

### 2.3 知识库运营能力

- 支持知识库分类，例如政策规则、市场流程、案例经验、问答资料。
- 支持文档状态管理，例如待处理、处理中、可用、失败、下线。
- 支持知识库配置差异化，例如不同知识库使用不同检索参数。

### 2.4 电力交易场景专项字段

- 增加省区字段。
- 增加市场类型字段。
- 增加业务主题字段。
- 增加发布日期、生效日期、失效日期。
- 增加发文机构和适用范围。

### 2.5 结构化数据接入准备

- 预留交易价格、成交量、市场参数等工具查询能力。
- 未来对接数据库/API，而不是只依赖文档检索。
- 业务上区分“知识问答”和“数据查询”。

## 3. 前端页面影响

- 聊天页：增加引用来源区、版本信息区、回答类型标识。
- 文档页：增加摄取状态、解析结果、失败原因、重试入口。
- 知识库页：增加知识库配置、启用版本、检索参数概览。
- 提示词配置页：增加版本列表、发布、回滚、差异对比。

## 4. 验收前置条件

- 统一主后端完成。
- MinIO / Milvus / OCR / 异步任务链路方案确认。
- 配置版本化设计确认。
- README 和架构文档验收通过。

## 5. 后续实施建议

- 第一批实施：来源展示、文档状态管理、知识库配置展示。
- 第二批实施：版本展示、回答类型分流、规则类直接引用模式。
- 第三批实施：结构化数据查询、工具调用、业务流程自动化。
