# 向量数据库快速参考

## 🚀 快速启动

### 1. 启动向量数据库

```bash
# 使用 Docker Compose 启动 ChromaDB 和 Milvus
docker-compose -f docker-compose-vector-db.yml up -d

# 查看运行状态
docker-compose -f docker-compose-vector-db.yml ps
```

### 2. 设置 API Key

```bash
# Windows PowerShell
$env:OPENAI_API_KEY="sk-your-api-key-here"

# Linux/Mac
export OPENAI_API_KEY="sk-your-api-key-here"
```

### 3. 启动应用

```bash
cd power-trade-rag-api
mvn spring-boot:run
```

## ⚙️ 配置文件速查

### 切换到 ChromaDB

```yaml
rag:
  vector-database-type: chroma
  chromadb:
    host: localhost
    port: 8000
    collection-name: power-trade-knowledge
```

### 切换到 Milvus

```yaml
rag:
  vector-database-type: milvus
  milvus:
    host: localhost
    port: 19530
    username: your-username
    password: your-password
    collection-name: power_trade_knowledge
    database: default
```

### OpenAI 配置

```yaml
langchain4j:
  open-ai:
    api-key: ${OPENAI_API_KEY}
    embedding-model-name: text-embedding-ada-002
    language-model-name: gpt-3.5-turbo
    base-url: https://api.openai.com/v1
```

## 🔧 核心类说明

### 配置类

| 类名 | 功能 |
|------|------|
| `VectorDatabaseConfig` | 向量数据库配置（ChromaDB/Milvus） |
| `OpenAiConfig` | OpenAI 模型配置（Embedding + LLM） |

### 服务类

| 类名 | 功能 |
|------|------|
| `VectorStoreService` | 向量存储管理（添加、搜索、删除） |
| `RagCoreService` | RAG 核心流程（问答、文档处理） |
| `DocumentPreprocessingService` | 文档预处理（解析、分块） |
| `SimilaritySearchEngine` | 相似度检索引擎 |
| `AnswerGenerationService` | 答案生成服务 |

## 📝 API 接口

### 智能问答

```http
POST /api/chat
Content-Type: application/json

{
  "sessionId": "session-001",
  "query": "什么是电力中长期交易？",
  "kbId": "KB001"
}
```

### 上传文档

```http
POST /api/documents/upload
Content-Type: multipart/form-data

file: [文件]
kbId: KB001
```

### 删除文档

```http
DELETE /api/documents/{docId}?kbId=KB001
```

### 获取知识库统计

```http
GET /api/knowledge-base/{kbId}/stats
```

## 💡 使用技巧

### 1. 批量向量化

```java
List<TextSegment> segments = ...;  // 多个文本片段
vectorStoreService.addTextSegments(segments, kbId);  // 批量添加
```

### 2. 按知识库搜索

```java
// 在指定知识库中搜索
List<EmbeddingMatch<TextSegment>> results = 
    vectorStoreService.searchByKnowledgeBase(query, 5, kbId);
```

### 3. 删除向量

```java
// 删除单个文档的向量
vectorStoreService.deleteDocumentVectors(docId);

// 删除整个知识库的向量
vectorStoreService.deleteKnowledgeBaseVectors(kbId);
```

## 🔍 故障排查

### ChromaDB 无法访问

```bash
# 检查容器状态
docker ps | grep chromadb

# 查看日志
docker logs chromadb

# 重启服务
docker-compose -f docker-compose-vector-db.yml restart chromadb
```

### Milvus 连接失败

```bash
# 检查 Milvus 状态
docker ps | grep milvus

# 查看日志
docker logs milvus-standalone

# 检查依赖服务
docker-compose -f docker-compose-vector-db.yml ps etcd minio
```

### OpenAI API 调用失败

1. 检查 API Key 是否正确
2. 检查网络连接
3. 查看应用日志：`tail -f logs/application.log`

## 📊 性能参数

### 推荐配置

```yaml
rag:
  document:
    chunk-size: 300        # 分块大小
    chunk-overlap: 30      # 重叠大小
    embedding-dimension: 1536  # text-embedding-ada-002 维度
  
  vector-store:
    auto-create-collection: true  # 自动创建集合
```

### 检索参数

```java
private static final int DEFAULT_MAX_RESULTS = 5;           // 默认返回 5 个结果
private static final double MIN_SIMILARITY_SCORE = 0.6;    // 最低相似度阈值
```

## 🎯 最佳实践

1. **文档分块**：300-500 tokens 为宜，重叠 10-20%
2. **批量处理**：批量向量化减少 API 调用
3. **缓存结果**：缓存常见问题的答案
4. **定期清理**：删除无用文档释放存储空间
5. **监控性能**：关注检索延迟和准确率
